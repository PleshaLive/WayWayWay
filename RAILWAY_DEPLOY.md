# Инструкции по деплою на Railway с PostgreSQL

## 🚀 Развертывание на Railway

### 1. Подготовка к деплою

1. Установите новые зависимости локально:
```bash
npm install pg@^8.11.3
```

2. Убедитесь, что все файлы добавлены в git:
```bash
git add .
git commit -m "Added PostgreSQL database support"
git push
```

### 2. Настройка базы данных на Railway

1. **Войдите в Railway Dashboard** (railway.app)
2. **Откройте ваш проект**
3. **Добавьте PostgreSQL базу данных:**
   - Нажмите "New" → "Database" → "Add PostgreSQL"
   - Railway автоматически создаст базу данных и переменную окружения `DATABASE_URL`

### 3. Переменные окружения

Railway автоматически создаст:
- `DATABASE_URL` - строка подключения к PostgreSQL
- `NODE_ENV` - установится в "production"

### 4. Что произойдет при первом запуске

1. **Автоматическая миграция:** При первом запуске система:
   - Создаст необходимые таблицы в PostgreSQL
   - Перенесет все данные из вашего `data.json` файла в базу данных
   - Сохранит команды и игроков навсегда

2. **Персистентное хранение:** После этого:
   - Все новые команды и игроки будут сохраняться в PostgreSQL
   - Данные не будут теряться при обновлениях кода
   - Файловые загрузки (логотипы, фото) останутся в Railway Volumes

### 5. Проверка работы

После деплоя проверьте:

1. **Логи приложения:**
```
Railway Dashboard → Your Project → Deployments → View Logs
```

Должны увидеть:
```
Database tables initialized successfully
Migration from JSON completed
Данные загружены из базы данных: X команд, Y игроков
```

2. **Функциональность:**
   - Откройте админку: `https://your-app.railway.app/admin`
   - Проверьте, что все команды и игроки загружены
   - Добавьте тестовую команду/игрока
   - Сделайте redeploy - данные должны сохраниться

### 6. Откат (если что-то пошло не так)

Если нужно вернуться к файловому хранению:

1. Удалите PostgreSQL базу из Railway
2. Верните старую версию кода через git
3. Данные из `data.json` будут использоваться снова

### 7. Преимущества новой системы

✅ **Персистентность:** Данные не теряются при redeploy  
✅ **Надежность:** PostgreSQL - промышленная СУБД  
✅ **Масштабируемость:** Поддержка множественных подключений  
✅ **Бэкапы:** Railway автоматически создает бэкапы БД  
✅ **Миграция:** Все старые данные автоматически перенесутся  

### 8. Мониторинг

В Railway Dashboard можно:
- Просматривать метрики базы данных
- Делать SQL запросы к БД
- Настраивать автоматические бэкапы
- Мониторить подключения

---

## 🔄 Альтернативное решение (если PostgreSQL не подходит)

Если не хотите использовать PostgreSQL, можно использовать Railway Volumes для персистентного хранения файлов:

1. Оставьте текущий код с `data.json`
2. В Railway добавьте Volume:
   - Dashboard → Project → Settings → Volumes
   - Mount Path: `/app/data`
   - Переместите `data.json` в эту папку

Но PostgreSQL решение более надежное!
